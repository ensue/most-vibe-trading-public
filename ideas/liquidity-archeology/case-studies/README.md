# Case studies

Save filled-in worked examples here. Naming convention:

```
YYYY-MM-DD-symbol-timeframe-context.md
```

For example:

- `2026-05-03-us100-monthly-cohort-map.md`
- `2026-06-15-btc-weekly-2024-cycle-cohort-map.md`
- `2026-07-01-eurusd-daily-2025-trend-cohort-map.md`

Each case study should:

1. Use a chart saved under `journal/charts/YYYY-MM/` (link it in the file header)
2. Either fill the `templates/cohort-snapshot.md` template (full survivor map) or `templates/event-analysis.md` template (single filtering event deep-dive)
3. End with **falsifiable predictions** that can be tested
4. Include **self-criticism** — where might this be post-hoc rationalization vs real prediction?
5. **Optional but recommended:** copy any real-money-testable prediction into a separate file in `journal/calls/` per `system/edge_verification.md`

Target: complete **at least 10 case studies across at least 3 asset classes** before judging whether this framework actually generates edge.
